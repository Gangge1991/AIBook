**常用评价指标**
实际有200封邮件，100封垃圾邮件，100封正常邮件
实际\预测   垃圾邮件  正常邮件
垃圾邮件     TP=80    FN=20
正常邮件     FP=10    TN=90

accuracy=(TP + TN) / (TP + TN + FP + FN)
precision = TP / (TP + FP)
recall = TP / (TP + FN)
fpr(false positive rate) = FP / (FP + TN) 
f1_score = 2 * (precision * recall) / (precision + recall)

**混淆矩阵**
                        Actual
                  Positive   Negative
         Positive  TP=560      FP=60 
predict      
         Negative  FN=50       TN=330
准确率
Accuracy=(TP+TN)/(TP+TN+FP+FN)=(560+330)/(560+60+50+330)=0.89 
精确率:预测为Positve中真实为Postive的比例
Precision = TP / (TP + FP)=560/(560+60)=0.95
召回率:真实为Positve中预测为Positve的比例
Recall=TP / (TP + FN)=560/(560+50)=0.96
F1-score：
F1=2/(1/Precision+1/Recall)=2*Precision*Recall/(Precision+Recall)=2*0.95*0.96/(0.95+0.96)=0.97

**uv是一个高性能的Python包和项目管理工具，它通过自动创建和管理项目特定的虚拟环境来简化开发。使用uv管理项目环境的主要步骤是：安装uv，使用uv venv创建虚拟环境（uv run可以自动创建），然后用uv add或uv pip install安装依赖，最后用uv run来运行脚本。为了确保环境的可复现性，可以使用uv lock或uv pip compile命令生成锁文件，然后通过uv sync在其他环境中安装相同的依赖。 
1. 安装uv
通过pip安装：pip install uv
其他安装方式：可以使用curl或官网提供的脚本安装。 
2. 项目环境管理
创建虚拟环境：
在项目根目录下运行uv venv .venv来创建一个名为.venv的虚拟环境。
如果执行uv run命令且没有虚拟环境，uv会自动创建并使用它。
若要指定Python版本，可以使用uv venv --python 3.x .venv。
激活虚拟环境：
Linux/macOS：source .venv/bin/activate
Windows：. .venv\\Scripts\\activate 
3. 包管理
安装依赖：
uv pip install <package_name>
uv add <package_name> (在pyproject.toml中添加依赖并安装)
从requirements.txt安装：uv pip install -r requirements.txt
升级包：uv pip upgrade <package_name>
生成锁文件：
uv lock或uv pip compile pyproject.toml -o uv.lock来固定当前环境的包版本。
在其他环境中同步依赖：uv sync**

 RESTful 优点
➢ 轻量，直接基于http，不再需要任何别的诸如消息协议
get/post/put/delete 为CRUD操作
➢ 面向资源，一目了然，具有自解释性。
➢ 数据描述简单，一般以 xml，json做数据交换。
➢ 无状态，在调用一个接口（访问、操作资源）的时候，可以不用考虑上下文，不用考虑当前状态，
极大的降低了复杂度。
➢ 简单、低耦合
